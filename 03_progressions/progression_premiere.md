# Progression annuelle - Première

## Principes
- 4h par semaine visées, progression sur 33h de base (modifiable selon l’horaire local).
- Une séquence par trimestre au minimum, avec projet terminal.

## Table de progression

### Séquence P1 — S01 : Représentation des données
- Période : 1
- Durée : 6 h
- Notions : listes, tuples, dictionnaires, fonctions simples
- Capacités : représenter et manipuler des structures, programmer des fonctions de calcul
- Documents nécessaires : cours élève, trace écrite, TD, TP, fiche méthode, aides progressives, corrigé, guide, éval, qcm, projet
- Évaluation : formative (1h) + sujet formatif QCM
- Projet associé : carnet de notes interactif (séq. de mini projet)
- Ressources réutilisées : `s01_representation_donnees/*`
- Ressources à créer : aucune pour ce lot

### Séquence P1 — S02 : Interaction homme-machine web (à créer)
- Période : 2
- Durée : 8 h
- Notions : événements, client-serveur, HTML/HTTP (initiation)
- Statut : à créer

### Séquence P1 — S03 : Algorithmique fondamentale
- Période : 2-3
- Durée : 12 h
- Notions : complexité élémentaire, recherche, tri
- Statut : à créer

### Séquence P1 — S04 : Langages et programmation
- Période : 3
- Durée : 8 h
- Notions : fonctions, structures de contrôle
- Statut : à créer

## Conformité programme
- Les séquences de 1re reprennent le schéma de 8 rubriques du référentiel, avec un bloc projet par trimestre.
