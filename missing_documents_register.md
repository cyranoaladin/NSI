# Registre des documents de séance manquants

- Statut : prototype non publiable.
- Ce registre documente les fichiers qui avaient été cités dans les plans de séances générés sans exister localement.
- Une ligne de registre ne rend jamais une absence acceptable : le contrôle échoue si le taux ou la criticité des absences dépasse les seuils.

## Règle

- Tout fichier cité dans une séance doit exister physiquement dans le dépôt.
- Les absences conservées au registre doivent avoir une décision explicite : créer, importer ou abandonner.
- Les documents haute priorité doivent avoir une date cible réelle au format ISO.
