# Checklist de revue scientifique

Cette checklist prépare une relecture scientifique humaine. Elle ne modifie aucun statut.

## Définitions

- Les définitions sont exactes et au niveau du programme.
- Les notations sont cohérentes dans toute la séquence.
- Les limites du programme sont respectées.

## Algorithmes et calculs

- Les traces d'exécution sont correctes.
- Les résultats numériques sont vérifiés.
- Les complexités annoncées sont justifiées au niveau attendu.
- Les pseudo-codes ne contredisent pas les corrigés.
- Les exemples SQL, graphes, arbres, réseaux, DP et Boyer-Moore sont cohérents avec leurs données.

## Code et données

- Les starters ne contiennent pas la solution complète.
- Les corrigés professeur passent les tests.
- Les tests couvrent cas nominal, cas limite et entrée invalide quand c'est pertinent.
- Les fichiers de données ne contiennent pas de données personnelles.
