# Plan projets NSI Première - Tunisie 2026-2027

- Statut : prototype annuel non publiable.
- Niveau : premiere
- Total horaire NSI estimé : 140 h
- Total projets : 43 h
- Ratio projet : 30.7 %
- Référence calendrier : `calendar_2026_2027_tunisia.md`.
- Décision : les projets structurent l'année mais ne rendent aucune ressource publiable.

## Principes

Les projets occupent plus d'un quart de l'horaire. Ils ne remplacent pas les cours, TD, TP et évaluations : ils servent à réinvestir les notions dans des livrables progressifs, avec carnet de bord et données fictives. Les données personnelles réelles sont interdites.

## Projet intermédiaire

- Période : fin janvier, avant Ramadan.
- Durée cumulée : environ 17 h de projet déjà engagées.
- Sujet recommandé : application locale d'exploration de données fictives avec conversions, types construits, CSV et tests.
- Livrables : code Python, tests, fichier de données fictives, notice utilisateur, carnet de bord, courte démonstration.
- Critères : fonctionnement observable, fonctions découpées, choix de structures justifié, gestion de cas limites, données non personnelles.
- Différenciation : version minimale en console ; version standard avec recherche et tri ; version experte avec fusion de tables et visualisation simple.

## Projet final

- Période : avril à juin, avec finalisation en P14.
- Sujet recommandé : mini-service local combinant données, algorithmes, interface Web simple et documentation.
- Version minimale : lire des données fictives, offrir une recherche, produire un résultat expliqué.
- Version standard : ajouter tri, fusion ou classification k-NN, tests et page Web locale.
- Version experte : ajouter analyse de complexité, gestion robuste des erreurs et protocole de test argumenté.
- Livrables : dossier projet, code, tests, guide utilisateur, fiche RGPD, carnet de bord, support de soutenance.

## Carnet de bord

Chaque élève conserve une trace courte mais régulière : objectif de séance, travail réalisé, blocage rencontré, aide reçue, test ajouté, décision prise. Le carnet sert à l'oral et à distinguer contribution individuelle et travail de groupe.

## Soutenance

- Durée indicative : 5 minutes de présentation et 3 minutes de questions en Première.
- Attendus : problème traité, données utilisées, choix de représentation, algorithme principal, démonstration, limites.
- Place de l'oral : micro-oraux dès P01, oral intermédiaire en janvier, soutenance finale en juin.

## Grille d'évaluation projet

| Critère | Indicateurs observables | Niveau fragile | Niveau attendu | Niveau avancé |
|---|---|---|---|---|
| Compréhension du problème | reformulation, exemples, limites | reformulation partielle | problème clair | limites discutées |
| Données et RGPD | données fictives, anonymisation simulée | données mal décrites | données sûres et documentées | justification fine des risques |
| Code | fonctions, lisibilité, tests | script monolithique | fonctions testées | architecture réutilisable |
| Algorithmique | choix, correction, cas limites | choix implicite | choix expliqué | complexité discutée |
| Communication | carnet, démonstration, oral | trace irrégulière | démonstration claire | recul critique |

## Gestion des groupes

- Groupes de 2 ou 3 élèves.
- Rôles tournants : responsable tests, responsable documentation, responsable intégration.
- Contribution individuelle documentée dans le carnet de bord.
- Les groupes utilisent uniquement des données fictives ou explicitement anonymisées pour l'exercice.

## Remédiation et différenciation

- Élèves fragiles : cahier des charges réduit, fichiers de départ, tests guidés, points de contrôle fréquents.
- Élèves au rythme standard : autonomie progressive sur structure du code et documentation.
- Élèves avancés : extensions bornées, analyse de complexité, amélioration ergonomique, validation d'entrée.
